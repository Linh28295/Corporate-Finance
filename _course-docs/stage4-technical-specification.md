# Stage 4: LLM-Drafted Technical Specification

**Weight:** 20% of project score (70% deliverable / 30% presentation)
**Deliverable:** Technical specification (`.md`) drafted with an LLM, plus a prompt log entry

---

## Overview

Use an LLM to draft a formal technical specification that fully defines both the Excel ratio model and the analytical work to be done on your selected company. The spec must be precise enough that an LLM with no prior context can produce a correct, comprehensive ratio analysis.

## Deliverable

Saved to `docs/specs/` as `YYYY-MM-DD-{lastname}-{company-slug}-spec.md`

## Required spec components

### Part A — Model Specification
1. Scope & Objective
2. Model Architecture
3. Data Inputs (values stated numerically from Stage 3)
4. Named Range Conventions
5. Derived Inputs
6. Ratio Definitions & Formulas
7. Validation Rules

### Part B — Analysis Specification
8. Analysis Requirements
9. Du Pont Decomposition
10. Strategic Recommendation Requirements
11. Output Format

## Rubric

| Criterion | % of deliverable |
|-----------|-----------------:|
| Model spec — Data & Structure (Part A, items 1–5) | 25% |
| Model spec — Ratios & Validation (Part A, items 6–7) | 25% |
| Analysis spec (Part B, items 8–11) | 25% |
| Spec craft + prompt log quality | 25% |
