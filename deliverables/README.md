## Naming convention

```
YYYY-MM-DD-{company-slug}-{descriptor}.md
```

**Examples:**
- `2026-05-16-coca-cola-llm-raw.md` — raw LLM output
- `2026-05-16-coca-cola-final-analysis.md` — evaluated final version
- `2026-05-16-coca-cola-peer-comparison.md` — extra credit peer analysis

## What belongs here

- **Stage 5 raw LLM output** — unedited response from executing the Stage 4 spec
- **Stage 5 evaluated final analysis** — edited, annotated, and corrected version with own judgement applied
- **Spec retrospective** — inside final analysis or as separate file
- **Prompt log** — consolidated record of all LLM prompts across stages
- **Extra credit outputs** — peer comparison, multi-year analysis, or other extensions

## What does NOT belong here

- Working drafts — keep in `docs/` or `analysis/`
- Source data — belongs in `data/`
- Excel models — belongs in `models/builds/`
